Travel Pro — Trip Overview Page Specification
This is the most important view in the app. It shows the user their generated travel itinerary with all stops, activities, flights, visa info, weather, budget, and an interactive map.

Context: This page is shown after the AI generates an itinerary. The data below is a real 22-day sample trip across Japan, Vietnam & Thailand for 2 travelers with a €10,000 budget in "comfort" travel style.

Brand: Primary teal (#0D7377), Accent coral (#E85D4A), Inter font, rounded cards (12px), pill-shaped badges. Clean, modern travel aesthetic.

Key interactions: Tab switching (Itinerary / Essentials / Budget / Map), day accordion expand/collapse, packing list checkboxes, budget adjustment buttons.

Data types available: Route (cities with lat/lng/days), Day-by-day activities (with tips, food recs, costs), Budget breakdown (by category + by city), Visa requirements per country, Weather per city, Flight legs (price, duration, airline), Packing list (auto-generated from weather/destinations).

DATA REFERENCE — These are the TypeScript interfaces and sample JSON for all data displayed on this page. Use this to understand what fields are available when designing variants.

TypeScript Interfaces

interface CityStop {
  id: string;           // "tokyo"
  city: string;         // "Tokyo"
  country: string;      // "Japan"
  lat: number;          // 35.68
  lng: number;          // 139.69
  days: number;         // 5
  countryCode: string;  // "JP"
  iataCode?: string;    // "NRT"
}

interface DayActivity {
  name: string;         // "Senso-ji Temple"
  category: string;     // "culture" | "food" | "explore" | "nature" | "art" | "nightlife" | "transport" | "adventure" | "wellness"
  icon: string;         // emoji, e.g. "⛩️"
  why: string;          // "Tokyo's oldest temple..."
  duration: string;     // "2h"
  tip?: string;         // "Visit before 9am..."
  food?: string;        // "Try melon pan..."
  cost?: string;        // "Free" | "€12" | "€20–30"
}

interface TripDay {
  day: number;          // 1
  date: string;         // "Oct 1"
  city: string;         // "Tokyo"
  activities: DayActivity[];
  isTravel?: boolean;         // true for transit days
  travelFrom?: string;       // "Tokyo"
  travelTo?: string;         // "Kyoto"
  travelDuration?: string;   // "2h 15min (Shinkansen)"
}

interface TripBudget {
  flights: number;        // 2500
  accommodation: number;  // 3500
  activities: number;     // 800
  food: number;           // 1200
  transport: number;      // 400
  total: number;          // 8400
  budget: number;         // 10000 (user's stated budget)
}

interface VisaInfo {
  country: string;
  countryCode: string;    // "JP"
  requirement: "visa-free" | "visa-on-arrival" | "e-visa" | "eta" | "visa-required";
  maxStayDays: number;
  notes: string;
  label: string;          // "Visa-free (90 days)"
  sourceUrl: string;      // link to official immigration site
}

interface CityWeather {
  city: string;
  temp: string;           // "22°C"
  condition: string;      // "Partly cloudy"
  icon: string;           // emoji
}

interface ItineraryFlightLeg {
  fromCity: string;       // "Frankfurt"
  toCity: string;         // "Tokyo"
  fromIata: string;       // "FRA"
  toIata: string;         // "NRT"
  departureDate: string;  // "2026-10-01"
  price: number;          // 680 (EUR)
  duration: string;       // "12h 30m"
  airline: string;        // "Lufthansa"
}

interface Itinerary {
  route: CityStop[];
  days: TripDay[];
  budget: TripBudget;
  visaData?: VisaInfo[];
  weatherData?: CityWeather[];
  flightLegs?: ItineraryFlightLeg[];
}

Key Design Requirements

  • Mobile-first responsive — must work beautifully on phone screens
  • Dense but scannable — 22 days of content needs clear visual hierarchy
  • Actionable — each activity has a "why" so users understand the recommendation
  • Travel days are visually distinct — different badge color, transport banner
  • Budget awareness — always show cost context (per activity, per city, total)
  • City grouping — days should be visually grouped by city stop
  • Progressive disclosure — accordion/collapse pattern to manage information density
  • Pro tips & food recs — these are a key differentiator, should be prominent
  • Flight overview — all legs visible at a glance with prices and times
  • Map integration — interactive map showing the full route with city pins
