Flights 3

POST
/api/v1/trips/[id]/flights Searches for real flight options between cities in the trip so users can compare prices and times. Optional

Developer

On-demand flight search for a trip leg. Calls SerpApi with optional filters (non-stop, max price). Returns up to 5 options per leg. Requires trip access (owner or guest).

PM

Searches for real flight options between cities in the trip so users can compare prices and times.

Auth: Optional (supports guests)
POST
/api/v1/trips/[id]/optimize Finds the best flight prices across the entire trip route and shows users how much the whole journey would cost. Required

Developer

Runs SerpApi flight price optimization across the full itinerary route. Returns a FlightSkeleton with per-leg pricing and a baseline total. Client is responsible for persisting the result.

PM

Finds the best flight prices across the entire trip route and shows users how much the whole journey would cost.

Auth: Required (trip owner)
GET
/api/v1/flights/book When a user clicks "Book" on a flight, this sends them to the airline's booking page. Optional

Developer

Returns HTML, not JSON. Resolves a flight booking token via SerpApi, logs an affiliate click (best-effort, non-blocking), and returns an auto-submitting HTML form that POSTs to the airline. This preserves browser context (cookies, session, referrer) that airlines require. Handles SerpApi rate limits with 429. See TRA-80 for planned revisit.

PM

When a user clicks "Book" on a flight, this sends them to the airline's booking page in a way that actually works (airlines are picky about how you arrive). We track the click for analytics.

Auth: Optional

Booking & Affiliate Tracking 2

GET POST PATCH
/api/v1/trips/[id]/booking-clicks Part of the trip preparation checklist. Users can confirm bookings or manually add bookings made elsewhere. Required

Developer

Manages booking confirmations in the trip preparation checklist. GET fetches existing clicks for a trip. POST creates a manual booking record (user booked outside the app). PATCH confirms or denies whether a click resulted in an actual booking.

PM

Part of the trip preparation checklist. Users can confirm "yes, I booked this flight/hotel" or manually add bookings they made elsewhere. Helps track what's done vs. what still needs booking.

Auth: Required (trip owner)
GET
/api/v1/affiliate/redirect When a user clicks a link to Skyscanner, Booking.com, or GetYourGuide, this logs the click and sends them to the partner site. None

Developer

Logs an affiliate click (provider, city, IP hash, optional metadata) to the affiliateClick table, then 302 redirects to the partner URL. Domain whitelist enforced. Used by buildTrackedLink() in frontend components.

PM

When a user clicks a link to Skyscanner, Booking.com, or GetYourGuide, this logs the click for analytics and sends them to the partner site. Works for non-logged-in users too.
